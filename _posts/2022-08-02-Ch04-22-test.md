# 오답 정리
4-1) 다음의 문장들을 조건식으로 표현하라.  
(2) char형 변수 ch가 공백이나 탭이 아닐 때 true인 조건식  
  오답: ch!=' ' || ch!='  ' // 말 그대로 탭을 눌러버림.   
  정답: ch!=' ' && ch!='\t' // 탭 = \t  
  
(8) 문자열 참조변수 str이 "yes"일 때 true 인 조건식.  
  오답: str == "yes"  
  정답: str.equals("yes")  
  // 변수에는 기본형(char, int, double ...)이 있고 그것을 제외한 변수를 참조형 이라고 한다.  
    equals 는 데이터 값을 비교하는 메소드 이며, == 은 주소값을 비교해주는 비교 연산자 이다.  
    따라서 참조형 변수를 비교할 때에는 == 를 사용할 수 없으며, 반대로 기본형 변수를 비교할 때에도 equals를 사용할 수 없다.

4-2) 1부터 20까지의 정수 중에서 2 또는 3의 배수가 아닌 수의 총합을 구하시오.  
내가 쓴 코드:
```java
class Ex2_1 { 
	public static void main(String args[]) { 
		//1~20까지의 정수 중에서 2또는 3의 배수가 아닌 수의 총합을 구하시오.
		int sum = 0, i = 1;
		for(i=1; i<=20; i++) {
			if(i%2==0 || i%3==0)
				continue;
			sum = sum + i;
			System.out.println("sum="+sum);
		}
	} 
}
```
정답이긴 하지만 예시 정답에 비하면 간결하지 못한 코드.

4-5) while문의 중첩에서 유의해야할 것  
오답:
```java
		int i = 0;
    int j = 0;
		while(i <= 10) {
			while(j <= i) {
				System.out.print("*");
				j++; //* ** j=3
			}
			System.out.println();
			i++;
		}
```
정답:
```java
		int i = 0;
		while(i <= 10) {
		int j = 0;
			while(j <= i) {
				System.out.print("*");
				j++; //* ** j=3
			}
			System.out.println();
			i++;
		}
```
j선언을 i선언과 같이 했을 때와(오답) j선언을 while문 안에서 했을 때(정답)에 따라서 결과가 전혀 달라졌다.  
오답의 경우  
```
*
*
*
*
*
*
*
*
*
*
```
정답의 경우
```
*
**
***
****
(...)
**********
```
카페에서 찾아본 결과 'i에 대한 반복이 한번 이루어질 때 마다 j를 0으로 초기화해주지 않으면 중첩된 while 문에 의해 j의 값은 무조건 i의 값과 동일해집니다.' 라는 것을 알게 되었다.  
잘 와닿지 않아서 for문으로 구구단 코드를 만든 것을 while문으로 구현해봤더니  
이번에도 j를 while문 밖에서 선언했을 때와 while문 안에서 선언했을 때 (즉, j를 0으로 초기화해줬을 때)의 결과가 전혀 다르게 나왔다.  
전자의 경우 구구단이 1단 즉, 1x1=1 1x=2=2 (...) 1x9=9 까지 밖에 안나왔고  
후자의 경우 구구단이 1~9단까지 정상 출력됐다.  
사실 왜 이런 상황이 발생하는지 잘 이해되진 않지만 직접 내가 확인해봤으므로 납득하기로 했다.
